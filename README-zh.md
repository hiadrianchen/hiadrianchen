> English: [README.md](README.md)

我在做一些偏实用的 AI 工具。我希望这些工作流和 Agent 能反复运行，有明确的评审检查点，并且在上下文出问题时还能恢复，而不是只做一次性的 demo。

## 公开项目

- **[challenge-plans](https://github.com/hiadrianchen/challenge-plans)** —— 一个多 Agent 工具，会在你执行 plan 或 spec *之前*，先做一轮对抗式评审，提前暴露站不住的假设、缺失的证据和交接风险。它跑在你本机已经登录的编码 CLI 上（Claude Code、Codex），不需要 API key。我关于"可靠的多 Agent 系统"的很多想法，都是在这个项目里变成可以真正跑起来的东西。

## 我关注的方向

- **不会自我欺骗的多 Agent 系统** —— 跨模型家族验证、用证据而不是票数说话、防住虚假共识和过早交接。
- **个人 AI 操作系统** —— 把笔记、任务、内容和长期运行的工作，沉淀成能活过单次对话的文件、队列和记忆。
- **可延续的 Agent 上下文** —— 版本化的记忆和共享上下文模式，让 Agent 重新接手时不用从头摸索基本信息。
- **知识沉淀** —— 把散落的输入（文章、视频、仓库、笔记）变成可检索、可复用的东西。

有些实验目前是 private，因为它们依赖个人上下文。等它们足够通用、可以脱离我的个人上下文运行时，我会把可复用的部分拆成公开工具——challenge-plans 就是第一个。

## 我的工作方式

我在意那些决定“一个 Agent 系统能不能用第二次”的部分：清晰的事实来源、可恢复的长任务、隐私边界，以及人工评审关卡。大致分工：

- 脚本负责抽取、清洗和校验；
- Agent 负责判断、综合和补救；
- 关键决策由人来拍板。
